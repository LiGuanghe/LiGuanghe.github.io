## 分享: 从跟大妈做项目学到的.
+ 项目领头人要做的: 筛选工具和思路
+ 项目各人员要做的: 领会领头人文件思想, 具体实现. 
+ 所有人员要做的: 将自己的思路和行动可视化. 
    + 即:    
```
 ipynb -> try -> src.
             \> doc.
```
    + 即:  git 和 github 
    + 即: 怼圈的最小工具链
    + 即: 怼圈的最小信息环
    + 即: Markdown 下的附带各种连接的 README 和文档.
eg: 大妈在ipynb里面关于数据清洗/整理/可视化的几个 ipynb 
eg: 大妈在src里面沉淀的脚本
eg: 大妈设置的文件层级

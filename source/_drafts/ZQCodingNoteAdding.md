### 需求到功能之间, 拆解 -> 想法 -> 实现 -> 调试 -> 拆解 ... 最后实现整个功能. 

之前不断强调的 MVP, 输入 -> 处理 -> 输出.

```
   代码 -> print -> 结果 -> 符合 -> #print -> git ->  下一个小想法
     ~             |
     |             v
     <------------不符合
```

if else 是编程的本质, 让计算机在面临选择的时候, 知道怎么进行下去. 每个脚本里的最小样本: 

```
if xx():
    print ...
else:
    pass
```

不断调试好,被注释掉的 print:

```
if xx():
    #print ...
else:
    if yy():
    print ...
    else: 
        pass
        ...
```

这样不断增加 if else 的过程, 就是不断从小到大实现想法, 最终实现功能的过程.
